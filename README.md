# ERP 시스템

회사 내부망에서 사용할 ERP 시스템이다.

엑셀, 카카오톡, 하이웍스 메일 중심으로 처리되던 업무를 단계적으로 전산화하는 것을 목표로 한다.

1차 목표는 자재관리와 재고관리 업무의 전산화이며, 이후 발주, 견적, 결재, 생산, 구매신청 등 업무 모듈을 순차적으로 확장한다.

## 프로젝트 방향

- ERP를 처음부터 모든 기능이 포함된 대형 시스템으로 만들지 않는다.
- 자재관리부터 시작하여 업무 모듈을 단계적으로 확장한다.
- 추후 확장이 가능하도록 모듈형 구조로 설계한다.
- 현재 단계에서는 AI 기능을 포함하지 않는다. 자재관리 MVP 안정화 이후 별도 프로젝트로 검토한다.

## 사용자 규모

- 현재 예상 사용자: 15명 ~ 20명
- 향후 대비: 동시 사용자 100명까지 고려

## 1차 개발 범위 (MVP)

- 자재관리
- 입고관리
- 출고관리
- 현재고관리
- 입출고이력관리
- 사용자 및 권한관리
- 엑셀 다운로드
- 변경이력 관리

## 향후 확장 범위

- 발주관리
- 견적관리
- 결재관리
- 생산관리
- 구매신청
- BOM 관리
- 작업지시
- 완제품 입출고 물류관리
- 은행 연동
- 홈택스 연동
- 메일 및 알림 시스템

## 현재 제외 범위

아래 기능은 현재 단계에서 개발하지 않는다. 단, 추후 확장 가능하도록 모듈형 구조로 설계한다.

- AI 기능
- 은행 연동
- 홈택스 연동
- 자체 메일 서버
- 완제품 대여 물류시스템
- 회계 기능
- 급여 기능

## Django 선택 이유

내부 ERP 시스템에 적합한 다음과 같은 특성을 가진다.

- 관리자 페이지 기본 제공
- 사용자 인증 / 그룹 / 권한 기능 기본 제공
- Django ORM으로 DB 모델 관리 가능
- Django Migration으로 DB 변경 이력 관리 가능
- Django REST Framework로 React 프론트엔드와 API 연동 가능
- CRUD가 많은 내부 업무 시스템에 적합
- 자재관리, 발주관리, 결재관리, 생산관리로 확장하기 좋음

## 전체 운영 구조

```
사용자 PC
    ↓
React / Refine / Ant Design
    ↓
Nginx / HTTPS
    ↓
Django REST Framework
    ↓
PostgreSQL
```

### 비동기 작업 구조

```
Django
    ↓
Celery
    ↓
Redis
    ↓
엑셀 생성, 알림 발송, 대량 처리 작업
```

## Django Admin 활용 방향

Django Admin은 최종 사용자용 ERP 화면이 아닌, 관리자 및 개발자 보조 도구로 사용한다.

### Django Admin 사용 대상

- 사용자 관리
- 권한 관리
- 자재 마스터 확인
- 거래처 확인
- 창고 확인
- 공통코드 확인
- 데이터 점검
- 운영자 긴급 수정

실제 사용자용 ERP 화면은 React / Refine으로 개발한다.
