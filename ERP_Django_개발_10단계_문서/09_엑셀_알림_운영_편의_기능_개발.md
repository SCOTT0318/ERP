# 9단계. 엑셀, 알림, 운영 편의 기능 개발

## 목적

기존 엑셀 중심 업무에서 ERP로 자연스럽게 전환할 수 있도록 편의 기능을 개발한다.

현업 사용자가 ERP에 적응하려면 엑셀 다운로드, 검색, 알림, 변경이력 확인 기능이 중요하다.

## 엑셀 다운로드

대상 화면

- 자재 목록
- 거래처 목록
- 창고 목록
- 현재고 현황
- 입고 현황
- 출고 현황
- 입출고 이력
- 재고 조정 이력

원칙

- 대량 데이터는 프론트에서 생성하지 않고 Django 백엔드에서 파일 생성
- 다운로드 이력 저장
- 권한 있는 사용자만 다운로드 가능
- 대량 다운로드는 Celery 비동기 작업 검토

## 엑셀 업로드

초기에는 자재 마스터 업로드부터 지원한다.

업로드 대상

- 자재 마스터
- 거래처 마스터
- 창고 및 위치 마스터

업로드 처리 흐름

1. 엑셀 파일 업로드
2. 데이터 형식 검증
3. 중복 데이터 검증
4. 오류 데이터 리포트 생성
5. 정상 데이터만 반영
6. 업로드 결과 저장

검증 항목

- 필수값 누락
- 자재코드 중복
- 단위 오류
- 창고코드 오류
- 숫자 형식 오류

## Celery 활용

시간이 오래 걸리는 작업은 Celery로 분리한다.

대상 작업

- 대량 엑셀 다운로드
- 대량 엑셀 업로드
- 알림 일괄 발송
- 오래된 로그 아카이빙
- 정기 백업 보조 작업

구성

- Django
- Celery Worker
- Redis Broker

## 내부 알림

처음부터 자체 메일 시스템을 만들지 않는다.

1차에서는 ERP 내부 알림을 개발한다.

알림 대상

- 입고 등록 알림
- 출고 등록 알림
- 재고 부족 알림
- 재고 조정 알림
- 시스템 공지

알림 기능

- 알림 목록
- 읽음 및 안읽음
- 알림 상세
- 알림 삭제
- 사용자별 알림

## 메일 기능 확장 방향

자체 메일 서버 개발은 후순위로 둔다.

권장 순서

1. ERP 내부 알림
2. SMTP 메일 발송
3. 하이웍스 메일 연동 검토
4. 필요 시 자체 메일 시스템 검토

자체 메일 시스템은 다음 항목이 필요하므로 초기 개발 범위에서 제외한다.

- SMTP
- IMAP
- 첨부파일 처리
- 주소록
- 스팸 처리
- SPF
- DKIM
- DMARC
- 백업
- 보안

## 변경이력 조회

관리자가 주요 변경 이력을 확인할 수 있어야 한다.

조회 조건

- 기간
- 사용자
- 기능
- 작업구분
- 대상 데이터
- IP

저장 항목

- 사용자
- 작업일시
- 작업구분
- 기능명
- 대상 앱
- 대상 모델
- 대상 ID
- 변경 전 값
- 변경 후 값
- 접속 IP

## 접속 로그

사용자 접속 로그를 관리한다.

저장 항목

- 사용자 ID
- 로그인 시간
- 로그아웃 시간
- 접속 IP
- 브라우저 정보
- 성공 및 실패 여부

- 메뉴 관리
