# 8단계. 자재관리 핵심 기능 개발

## 목적

자재관리 ERP의 핵심 기능을 개발한다.

1차 MVP에서 실제 현업이 사용할 기능은 자재 등록, 입고, 출고, 현재고, 입출고 이력이다.

## 개발 대상 기능

- 자재 마스터
- 거래처 및 공급처 관리
- 창고 관리
- 위치 관리
- 입고 관리
- 출고 관리
- 현재고 관리
- 입출고 이력 관리
- 재고 조정

## Django 트랜잭션 처리

입고, 출고, 조정은 반드시 transaction.atomic으로 처리한다.

하나라도 실패하면 전체 rollback 처리한다.

처리 대상

- 입고 등록
- 입고 취소
- 출고 등록
- 출고 취소
- 재고 조정
- 창고 이동

## 동시 출고 방지

동시에 여러 사용자가 같은 자재를 출고할 수 있으므로 현재고 차감 시 동시성 처리를 한다.

처리 방식

- select_for_update 사용 검토
- 현재고가 출고수량 이상일 때만 차감
- 차감 대상이 없으면 출고 실패 처리
- DB 트랜잭션으로 처리

## Django Admin 활용

Django Admin에서는 관리자용으로 주요 모델을 등록한다.

대상

- Material
- Partner
- Warehouse
- WarehouseLocation
- StockBalance
- StockTransaction
- User
- Group
- AuditLog

주의사항

- Admin에서 재고 수량을 직접 수정하지 않도록 제한한다.
- 재고 수정은 반드시 재고조정 기능을 통해 처리한다.
