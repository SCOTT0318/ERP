# 5단계. 인증, 권한, 보안 구조 개발

## 목적

내부망 ERP라도 반드시 인증과 권한, 보안 구조를 적용한다.

ERP에는 회사의 자재, 재고, 입출고 이력 등 중요한 데이터가 저장되므로 내부망이라는 이유만으로 보안을 생략하면 안 된다.

## 인증 방식

Django Auth를 기본으로 사용한다.

React 프론트엔드와 API 연동을 위해 JWT 인증 또는 세션 인증 중 하나를 선택한다.

권장 구성

- Django Admin은 Django 기본 세션 인증 사용
- React ERP 화면은 JWT 인증 사용
- 비밀번호는 Django 기본 해시 시스템 사용
- 권한은 Django Group 및 Permission 기반으로 확장

## 로그인 기능

로그인 API

- POST /api/auth/login

토큰 갱신 API

- POST /api/auth/refresh

사용자 정보 API

- GET /api/auth/me

로그아웃 API

- POST /api/auth/logout

초기 권한 그룹은 다음과 같이 구성한다.

- 관리자
- 자재관리자
- 입고담당자
- 출고담당자
- 조회전용

- 자재 수정 가능

- 현재고 조회 가능
- 재고 조정 가능

입고담당자

- 입고 등록 가능
- 입고현황 조회 가능
- 현재고 조회 가능

출고담당자

- 출고 등록 가능
- 출고현황 조회 가능
- 현재고 조회 가능

- 조회만 가능

현장작업자

- 제한된 입고 및 출고 작업 가능
- 관리 기능 접근 불가

## 권한 적용 방식

권한은 프론트엔드와 백엔드 모두에서 처리한다.

프론트엔드

- 권한 없는 메뉴 숨김
- 권한 없는 버튼 숨김
- 화면 접근 제한

백엔드

- DRF Permission 적용
- API 호출 시 권한 검증
- 권한 없는 요청 차단
- 데이터 변경 권한 검증

중요한 것은 백엔드 권한 검증이다.

프론트엔드에서 버튼을 숨기는 것만으로는 충분하지 않다.

## Django Admin 보안

Django Admin은 강력하지만 접근 권한을 제한해야 한다.

관리 기준

- 관리자만 접근 가능
- 일반 사용자 접근 금지
- 강한 비밀번호 사용
- 퇴사자 계정 즉시 비활성화
- 운영 환경에서 Admin URL 변경 검토
- 접속 로그 저장
- HTTPS 적용

## 비밀번호 정책

- 비밀번호 평문 저장 금지
- Django 기본 비밀번호 해시 사용
- 초기 비밀번호 변경 유도
- 비밀번호 복잡도 정책 적용
- 퇴사자 계정 비활성화

## 접속 로그

사용자 접속 이력을 저장한다.

저장 항목

- 사용자 ID
- 접속 IP
- 로그인 시간
- 로그아웃 시간
- 로그인 성공 여부
- 로그인 실패 여부
- 브라우저 정보

## 변경이력 로그

주요 데이터 변경 시 반드시 이력을 저장한다.

저장 항목

- 사용자
- 작업일시
- 작업구분
- 기능명
- 대상 앱
- 대상 모델
- 대상 ID
- 변경 전 값
- 변경 후 값
- 접속 IP

대상 기능

- 자재 등록
- 자재 수정
- 자재 삭제
- 입고 등록
- 입고 취소
- 출고 등록
- 출고 취소
- 재고 조정
- 사용자 권한 변경
- 엑셀 다운로드

## 내부망 보안 기준

- DB 포트는 사용자 PC에서 직접 접근 불가
- Django 서버만 DB 접근 가능
- HTTPS 적용
- 관리자 페이지 접근 제한
- 서버 접근 계정 제한
- 정기 백업
- 로그 보관
- DEBUG는 운영에서 False
- SECRET_KEY 외부 노출 금지
