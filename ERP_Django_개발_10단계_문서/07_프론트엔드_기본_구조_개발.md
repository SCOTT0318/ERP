# 7단계. 프론트엔드 기본 구조 개발

## 목적

Refine + React + TypeScript + Ant Design 기반의 ERP 프론트엔드 구조를 개발한다.

ERP는 화면 수가 많고 CRUD 기능이 많으므로, 처음부터 확장 가능한 구조로 만든다.

백엔드는 Django REST Framework API를 제공하고, 프론트엔드는 해당 API를 호출한다.

## 확정 프론트엔드 스택

- Refine
- React
- TypeScript
- Ant Design
- Vite
- React Router
- Axios

## 선택 이유

- ERP처럼 등록, 조회, 수정, 삭제 화면이 많은 시스템에 적합
- Ant Design의 업무용 UI 컴포넌트 활용 가능
- Django REST Framework API와 연동하기 좋음
- 권한별 메뉴 및 버튼 제어가 가능
- 향후 발주, 견적, 결재, 생산관리 모듈 확장에 유리

## 화면 설계 원칙

### 1. 엑셀과 유사한 사용성

현재 업무가 엑셀 중심이므로 ERP 화면도 사용자가 쉽게 적응할 수 있어야 한다.

- 목록 중심 화면
- 검색 조건 제공
- 엑셀 다운로드 제공
- 필터 제공
- 정렬 제공
- 페이지네이션 제공

### 2. 서버 사이드 페이지네이션

데이터를 한 번에 모두 가져오지 않는다.

예시

- GET /api/materials/?page=1&page_size=50&search=볼트

검색, 정렬, 페이지네이션은 서버에서 처리한다.

### 3. 권한별 UI 제어

권한이 없는 사용자는 메뉴와 버튼이 보이지 않게 한다.

- 메뉴 권한
- 버튼 권한
- 화면 접근 권한

단, 최종 권한 검증은 Django 백엔드에서 처리한다.

### 4. 공통 컴포넌트화

반복되는 UI는 공통 컴포넌트로 만든다.

- 검색 영역
- 목록 테이블
- 등록 및 수정 폼
- 확인 모달
- 엑셀 다운로드 버튼
- 상태 태그
- 날짜 범위 선택
- 창고 선택
- 자재 선택

## Django API 연동 기준

React는 Django API를 직접 호출한다.

API 인증 방식

- Access Token 저장
- Refresh Token 처리
- 토큰 만료 시 재발급
- 권한 없음 응답 처리
- 로그인 만료 시 로그인 화면으로 이동

주의사항

- 비밀번호는 브라우저에 저장하지 않는다.
- 민감한 정보는 localStorage 저장을 최소화한다.
- API 권한 검증은 반드시 Django에서 처리한다.
